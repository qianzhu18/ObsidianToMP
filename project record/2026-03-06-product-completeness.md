# ObsidianToMP 产品完善度整理（2026-03-06）

## 1. 本次问题根因
- 外部主题/高亮下载失败的核心原因有两点：
1. Release 缺少 `assets.zip`，导致下载地址 404。
2. 插件下载地址只拼接 `1.0.0`，但仓库标签是 `v1.0.0`，标签格式不匹配。

1. 资源下载支持多地址兜底：`latest` + `v版本` + `纯版本`。
2. 首次启动自动尝试下载主题/高亮资源。
3. 下载失败时不再误显示“下载完成”，并给出真实失败信息。
4. 资源加载增加异常兜底，避免因资源问题导致插件整体不可用。
5. 缺失资源提示去噪，避免重复弹窗影响使用。

1. 已向 `v1.0.0` release 补齐 `assets.zip`。

- 文档层：
1. README 补充 BRAT 安装后的资源自动下载说明。
2. README 补充外部网络下载失败的排障步骤。

## 3. 当前 MVP 完整度评估
- 核心链路：`本地写作 -> 预览 -> 复制 -> (可选) 发草稿`：可用。
- 图床链路：支持 S3 兼容上传，能力可用；对象权限错误（ACL/403）需用户在云端配置修复。
- 主题高亮：具备自动下载 + 手动下载双路径，外部账号可安装后使用。
