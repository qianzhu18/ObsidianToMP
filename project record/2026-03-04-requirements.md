# Project Record - 2026-03-04

## 用户目标（最新）
1. 打造一款可在 Obsidian 中稳定使用的本地微信编辑发布插件。
2. 复制粘贴要无门槛可用，尽可能减少格式损耗。
3. 支持一键同步到微信公众号草稿箱。
4. 解决本地图片上传痛点：
   - 不依赖微信公众号上传链路时，也能把本地图片上传到用户自己的云端。
   - 配置尽量简单，填写桶/Endpoint/Key 等基础信息即可使用。
5. 项目开源并持续打磨，目标 50 stars。

## 当前已完成
- stable 分支建立并持续集成。
- 前端品牌从旧项目标识切换为 ObsidianToMP。
- 复制链路稳定性增强：浏览器剪贴板 + Electron 剪贴板双兜底。
- 代码块粘贴保真增强（保留换行/空行）。
- 新增 S3 兼容图床配置。
- 新增“上传图片到云端”按钮。
- 新增复制时自动上传云图（未选择公众号账号时）。

## 当前待优化
- 模板间复制仍存在少量边缘问题（列表、表格、Callout）。
- 需要增加回归测试文档与固定测试样例。
- 需要补充更完整的上手 Demo 与 FAQ。

## 产品方向原则
- Local-first（本地优先）
- 可回退（出错可手动复制/手动发布）
- 低配置（开箱能跑，进阶可配置）
- 可维护（stable 分支优先）
