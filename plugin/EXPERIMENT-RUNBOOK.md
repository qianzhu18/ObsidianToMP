# Obsidian 插件实验 Runbook（公众号 MVP）

## 1. 实验目标
- 验证 MVP 闭环：`预览 -> 复制` 与 `预览 -> 发草稿`。
- 验证当前迭代能力：
  - 多主题渲染（含新增 Raphael 主题）
  - 多端预览（手机/平板/桌面）
  - 公众号账号配置、测试与草稿同步

## 2. 本地准备
在源码目录执行：

```bash
cd "/Users/mac/Downloads/code/obsidian publish/note-to-mp-v200"
npm install
npm run build
```

构建产物：
- `main.js`
- `styles.css`
- `manifest.json`

## 3. 投入 Obsidian 实验（推荐软链接）
插件 ID 是 `note-to-mp`，目标目录应为：

`<你的Vault路径>/.obsidian/plugins/note-to-mp`

推荐方式（便于持续迭代）：

```bash
mkdir -p "<你的Vault路径>/.obsidian/plugins"
ln -sfn "/Users/mac/Downloads/code/obsidian publish/note-to-mp-v200" "<你的Vault路径>/.obsidian/plugins/note-to-mp"
```

然后在 Obsidian 中：
1. 关闭安全模式（如未关闭）
2. 第三方插件里启用 `NoteToMP`
3. 修改代码后执行 `npm run build`，然后在插件管理页关闭再启用一次插件

## 4. 公众号账号配置
在插件设置的“公众号信息”中按行填写：

`公众号名称|AppID|AppSecret`

注意：
- 现在 Secret 会掩码显示；保持掩码不改则沿用旧 Secret。
- 点击“测试公众号”必须通过，才能进入草稿发布验证。
- 若报 `40164`，说明当前出口 IP 不在公众号白名单，需要到微信公众平台配置。

## 5. MVP 验证用例

### 用例 A：多端预览
1. 打开一篇包含标题、正文、代码块、图片的 Markdown。
2. 在预览顶部切换：`手机` / `平板` / `桌面`。
3. 检查排版是否稳定（标题间距、图片宽度、代码块换行）。

通过标准：三档预览可切换，内容不重叠不裁剪。

### 用例 B：复制到公众号编辑器
1. 在桌面端 Obsidian 点击“复制”。
2. 打开微信公众平台新建图文，直接粘贴。
3. 检查样式、代码高亮、图片链接是否正常。

通过标准：可直接粘贴且结构完整。

### 用例 C：一键发草稿
1. 在预览中点击“发文章”。
2. 到公众号后台草稿箱确认文章生成。
3. 检查封面、正文、图片是否正确。

通过标准：草稿创建成功，无接口报错。

## 6. 回归检查（每次迭代后）
- 账号：保存后刷新设置页，账号仍可读取。
- 主题：切换 3 个以上主题，渲染无明显错位。
- 图片：本地图片与远程图片都能处理。
- 发布：连续操作“复制 -> 发文章 -> 发图文”无 token 异常。

## 8. 迭代节奏建议
- 小改动：`npm run build` 后立即回归用例 A + B。
- 发布链路改动：必须回归 A + B + C。
- 每晚收敛一次问题清单，按“阻断发布 > 体验问题 > 视觉细节”排序修复。
